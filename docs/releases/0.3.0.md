# qbvisor 0.3.0

Released July 19, 2026.

qbvisor `0.3.0` develops the original Quickbase API wrapper into a broader SDK for using Quickbase
as an application backend. The release keeps the established `QuickBaseClient` facade and its most
used method signatures while strengthening the transport, data movement, backup, and application
building workflows beneath it.

## Highlights

- Query complete table and report results into pandas DataFrames by default, with optional limits
  for callers that need bounded memory use.
- Upsert records in payload-aware batches and report complete, partial, and uncertain outcomes.
- Export stable Record ID-ordered CSV files and download attachments with bounded concurrency.
- Capture versioned application backups with manifests, attachment history, and integrity checks.
- Plan and apply declarative applications, tables, fields, formulas, and relationships while
  preserving durable Quickbase resource bindings.
- Use pooled connections, explicit timeouts, structured exceptions, conservative retries, and
  redacted diagnostics across the synchronous client.
- Inspect additional Quickbase resources including app events, roles, field usage, relationships,
  and record changes.

## Compatibility

Python 3.12 or later is required. Python 3.12, 3.13, and 3.14 are tested.

Existing `QuickBaseClient` method names and the public signatures of the primary query, report,
export, attachment, and upsert workflows remain available. Some corrected behavior is observable:

- `query_dataframe()` and `run_report()` return complete requested results when `top` is omitted.
- transport failures raise qbvisor exception types instead of raw `requests` exceptions;
- upserts expose partial and uncertain batch outcomes;
- attachment failures raise `QuickbaseBatchError` after independent downloads complete;
- collection endpoints preserve the documented Quickbase response shape.

Review [Upgrading from 0.2 to 0.3](../upgrading-to-0.3.md) before updating a production workflow.

## Installation

Install and pin the release from PyPI:

```bash
python -m pip install qbvisor==0.3.0
```

Or add it to a uv-managed project:

```bash
uv add qbvisor==0.3.0
```

## Operational boundaries

- Complete DataFrame and report reads are materialized in memory. Use `top` or CSV export for
  workloads that should remain bounded.
- Quickbase does not provide transactional application snapshots. Backups detect changes during
  capture but cannot guarantee a database-wide point-in-time snapshot.
- Declarative apply creates and updates declared resources. It does not delete undeclared
  resources.
- Mutations are not replayed after an uncertain connection failure.
- Backups and exports may contain sensitive application data and require appropriate storage
  controls.
